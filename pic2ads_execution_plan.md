# Pic2Ads — Execution Plan (v2)

> AI video ad platform at **pic2ads.io**. Turns a product photo into finished video ads across three creative modes. Python backend, Seedance 2.0 (via Kie.ai) as the workhorse video model, Veo 3 (via fal.ai) for optional cinematic polish, with Seedance 2.0 extend used for longer continuous shots, Claude for creative agents, GPT-4o for vision. Prompt architecture adapted and extended from the Recap AI Sora workflow.

This revision bakes in two hard constraints: **15-second max per generation** and **Seedance 2.0 extend as the long-video primitive**. Those constraints reshape the video pipeline, the continuity engine, and the cost model. Efficiency is the explicit design goal throughout.

---

## 1. Executive Summary

**Product.** Pic2Ads is a web app where a user uploads a product photo, picks a mode (UGC / Professional / TV), and receives platform-ready video ads in 3–20 minutes. Built on an agentic creative pipeline that mirrors how a real ad shop works — Casting Director, Brand Strategist, Screenwriter, Director, Cinematographer, Continuity Manager, Frame Designer, Video Generator, Editor — each as a tightly-scoped agent with typed I/O.

**What makes it different.**
1. **Three modes, one system**: UGC influencer, Professional UGC with character arc, and TV commercial — each its own agent graph, not a toggle.
2. **Extend-native duration strategy**: 15s constraint turned into a feature via Seedance 2.0 extend for continuous takes and deliberate cuts for multi-scene spots.
3. **Two-tier video stack**: Seedance 2.0 as the default cheap/fast workhorse; Veo 3 only when the shot demands extra cinematic polish.
4. **Continuity engine**: extend for within-take, Style Bible + first-frame chaining + character reference card for across-cut continuity.
5. **Scene graph editor**: regenerate a single shot (or single extension segment) without re-rendering the whole spot.
6. **Hook Lab**: generate 3-second openers cheaply on Seedance, A/B them, then commit to the full render with the winner locked in.
7. **Brand Lock**: upload brand bible once; every render honors palette, voice, forbidden claims.
8. **Ad-platform-ready exports**: one job produces 9:16 + 1:1 + 16:9 with per-platform safe zones.

**Engineering principle.** Cheapest path that meets the quality bar. Seedance 2.0 is the default; Veo 3 is earned by specific shot requirements (cinematic polish score above a threshold, or a Seedance failure to meet QA).

## 2. The Three Modes

Each mode is a distinct agent graph, duration strategy, and model routing default.

### 2.1 Mode A — UGC Influencer (10–15s, single shot)

Fits the Seedance sweet spot: one shot, one call, one creator, handheld aesthetic. Direct descendant of the inspiration workflow.

- **Duration:** 10, 12, or 15s. Single Seedance 2.0 generation.
- **Creator:** one persona, generated by the Casting Director.
- **Output:** 3 script variants (Excited Discovery / Casual Recommendation / In-the-Moment Demo) — 3 parallel Seedance jobs.
- **Delivery:** 9:16 primary, 1:1 optional crop.
- **Ship time:** 3–6 min end-to-end per run.

### 2.2 Mode B — Professional UGC with Character Arc (20–30s)

The wedge mode. UGC intimacy with a narrative spine (friction → product → resolution). Two execution modes inside Mode B, picked by the Director:

**B1 — Continuous Take (Seedance 2.0 + extend):** one character, one location, time compression shown via camera movement. 15s base + 15s extend = 30s. Native continuity because the last frame of shot 1 is the first frame of the extension. Best for intimate "day in the life" narratives.

**B2 — Cut-Based (Seedance × N):** 2–3 shots with hard cuts. Cheaper. Cuts signal location/time jumps (e.g., bedroom → bathroom → kitchen). Requires our full Continuity Manager to hold the character across cuts.

- **Duration:** 20s or 30s. User picks.
- **Director decides** B1 vs B2 based on the brief: same-location arc → B1; location-hopping arc → B2.
- **Output:** one hero spot + optional 10s/15s cutdowns auto-derived.
- **Ship time:** 5–12 min.

### 2.3 Mode C — TV Commercial (15s / 30s / 60s)

Full cinematic, multi-shot, multi-character, optional multi-location.

- **Structure:** 3–8 shots, each ≤15s (usually 3–10s). Hard cuts between shots.
- **Within-shot extension:** Seedance 2.0 extend used selectively for any single "hero shot" that needs to run long (e.g., a 20s continuous push-in on the product).
- **Cross-shot continuity:** our Continuity Manager (Style Bible + character reference card + first-frame chaining) — same discipline as B2 but over more shots.
- **User-in-the-loop:** Concept Generator returns 3 one-page treatments; user picks one before any video renders. Storyboard review step before shot rendering. Saves dramatic cost if the direction is off.
- **Output:** one hero spot + auto-derived 6s / 15s / 30s / 60s cutdowns in 16:9, 9:16, and 1:1.
- **Ship time:** 15–30 min.

### 2.4 What's shared

Product Intelligence, Brand Lock, Frame Designer, Video Model Router, Editor (captions + exports), QA Agent. The difference between modes is which agents get added (Concept Generator, Continuity Manager, Composer, VO) and which duration/model strategy is applied.

## 3. Duration Strategy (the 15-second reality)

This is the unlock. Rather than treat the 15s cap as a limitation, we make it a first-class architectural primitive.

### 3.1 Three duration patterns

Every piece of video in Pic2Ads is one of three patterns:

**Pattern 1 — Single-Gen (≤15s).** One provider call. Used for UGC and for any individual shot in a commercial. Default model: **Seedance 2.0**.

**Pattern 2 — Extend-Chain (>15s, continuous take).** Base 15s generation + one or more 15s extensions. Provider: **Veo 3 via fal.ai** (only Veo 3's extend gives frame-perfect continuity right now — confirm current capabilities before building). Used for Mode B1 and any hero shot in Mode C that needs to run long.

**Pattern 3 — Cut-Chain (>15s, with hard cuts).** N independent Single-Gen shots, stitched with cuts. Provider: **Seedance 2.0** per shot. Continuity held by our Style Bible + character reference. Used for Mode B2 and the default for Mode C.

### 3.2 Who picks the pattern

A new agent — the **Duration Planner** — runs after the Screenwriter and before any rendering. Its input: the script's beat sheet + polish config + budget tier. Its output: a list of "render units," each tagged with a pattern and target duration.

```python
class RenderUnit(BaseModel):
    sequence: int
    pattern: Literal["single_gen", "extend_chain", "cut_chain"]
    duration_s: int           # total seconds the unit must produce
    segments: list[Segment]   # for extend_chain / cut_chain, the 15s pieces

class Segment(BaseModel):
    order: int                # 0 = base, 1..N = extensions (for extend_chain)
    duration_s: int           # ≤ 15
    prompt: str
    reference_frame_url: str | None
    model_hint: VideoModel | None
```

The Duration Planner is a small deterministic agent (rules + one LLM call for the beat-to-segment mapping). It does not make creative decisions — those are already made upstream.

### 3.3 Why this matters for efficiency

- **Cost floor.** A UGC ad is one Seedance call. A 30s TV spot is 3–6 Seedance calls. Extend is reserved for cases that genuinely need it, not the default.
- **Failure isolation.** If segment 2 of an extend-chain fails QA, we regenerate just that segment, not the whole 30s piece.
- **Parallelism.** Cut-chains render in parallel. Extend-chains are necessarily sequential (segment 2 depends on segment 1), so the planner marks them for priority scheduling.
- **Cache reuse.** Style Bible, persona card, and first frames are computed once and reused across all segments of a unit.

## 4. The Prompt Architecture We're Adapting

### 4.1 The five principles from the Recap workflow (canonized)

Every creative agent prompt in Pic2Ads obeys these:

1. **Role + Goal + Non-Goal stated upfront.** The non-goal is as important as the goal — stops LLM drift.
2. **Typed input contract.** Every input is declared with a Pydantic field, no ambiguous interpolation.
3. **Rigid output structure.** Not "describe X" — "I. Section → named fields." Forces specificity.
4. **Positive/Negative framing.** "What we WANT" vs "What we AVOID." LLMs learn style from contrast faster than from description.
5. **Temporal precision.** Second-by-second breakdowns where they matter (shot design, dialogue pacing).

### 4.2 How we extend the pattern

- **Decomposition.** Inspiration has 2 big prompts; we split into ~15 specialist agents with typed I/O between them. Easier to eval, cheaper to iterate, debuggable per-agent.
- **Mode-conditioned overlays.** One base prompt per agent, plus a mode overlay (UGC / Pro / TV) that adjusts WANT/AVOID lists and polish constraints.
- **Polish vector.** `{handheld, lighting_quality, color_grade, audio_quality, cuts_per_second, vfx_budget}` — a shared context object every agent reads so the whole pipeline is on the same aesthetic page.
- **Prompts are versioned files**, not strings in a DB. `/prompts/casting_director/v3.jinja`. Every version runs through an eval suite before promotion.
- **Replayability.** Every agent output is stored with an input hash; regens reuse upstream outputs.

### 4.3 The standard agent skeleton

```
[ROLE]        one sentence
[GOAL]        one sentence
[NON-GOALS]   bulleted
[INPUTS]      typed, rendered from Pydantic
[OUTPUT]      JSON schema
[STYLE RULES] WANT / AVOID (mode-conditioned)
[EXAMPLES]    1–2 few-shot, only for tricky agents
```

### 5.1 Example: Casting Director (adapted from the Recap prompt)

```python
class Persona(BaseModel):
    name: str
    age: int = Field(ge=16, le=90)
    gender: Literal["female", "male", "nonbinary"]
    location_descriptor: str
    occupation: str
    appearance: str
    hair: str
    clothing_aesthetic: str
    signature_details: list[str]
    traits: list[str] = Field(min_length=5, max_length=7)
    demeanor: str
    speaking_style: str
    hobbies: list[str]
    values: list[str]
    pain_points: list[str]
    home_environment: str
    credibility_rationale: str = Field(max_length=300)

class CastingDirectorAgent:
    prompt = "prompts/casting_director/v3.jinja"

    async def run(self, intel: ProductIntel, brand: BrandConstraints | None) -> Persona:
        system = render(self.prompt, intel=intel, brand=brand)
        return await llm.structured(
            model="claude-sonnet-4-6",
            system=system,
            parts=[image_part(intel.product_image_url)],
            response_model=Persona,
        )
```

The Jinja template embeds the five-part structure (Identity / Look / Vibe / Context / Why), explicit non-goals ("no scripts, no hooks, no concepts"), and brand constraints when available.

### 5.2 Example: Video Model Router (concrete logic)

```python
def route_segment(seg: Segment, unit: RenderUnit, bible: StyleBible | None) -> ModelPlan:
    if unit.pattern == "extend_chain" and seg.order > 0:
        return ModelPlan(model=VideoModel.SEEDANCE_2, mode="extend", source_segment=seg.order - 1)

    if unit.pattern == "extend_chain" and seg.order == 0:
        return ModelPlan(model=VideoModel.SEEDANCE_2, mode="base")

    needs_cinematic = bible and bible.polish.color_grade > 0.7
    needs_veo = needs_cinematic or seg.requires_complex_physics

    primary = VideoModel.VEO_3_FAST if needs_veo else VideoModel.SEEDANCE_2
    fallback = [VideoModel.SEEDANCE_2, VideoModel.VEO_3_FAST, VideoModel.KLING_2]
    fallback = [m for m in fallback if m != primary]

    return ModelPlan(model=primary, mode="base", fallback=fallback)
```

Principle: **default Seedance, earn Veo 3**. This keeps the cost floor low and only pays for Veo when the shot genuinely needs cinematic quality.

### 6.1 Video provider integration notes

**Kie.ai (Seedance 2.0):** HTTP API with async create + poll pattern. Wrap in a single `SeedanceClient` with `create_base(prompt, image, duration)` and `extend(source_video_url, prompt, duration)` plus `poll(job_id)` → `status/url`, with exponential backoff. Validate webhook support (if offered) to avoid naked polling at scale.

**fal.ai (Veo 3):** We use the base generation endpoint for selective cinematic shots. fal supports both sync and async; we use async with webhooks.

Both wrappers implement the same `VideoGenerator` protocol so the router is model-agnostic.

```python
class VideoGenerator(Protocol):
    async def generate(self, seg: Segment, plan: ModelPlan) -> VideoJob: ...
    async def poll(self, job: VideoJob) -> VideoResult: ...
```

### 7.1 Temporal workflow (annotated)

```python
@workflow.defn
class GenerateAdWorkflow:
    @workflow.run
    async def run(self, req: AdRequest) -> AdResult:
        intel = await wf.execute_activity(product_intel, req)
        brand = await wf.execute_activity(brand_strategy, req.brand_id, intel)

        if req.mode == "ugc":
            persona = await wf.execute_activity(casting, intel, brand)
            scripts = await wf.execute_activity(screenwriter_ugc, persona, intel, brand, count=3)
            units = [await wf.execute_activity(duration_plan, s) for s in scripts]
            results = await asyncio.gather(*[
                wf.execute_child_workflow(RenderUnitWorkflow, u) for u in units
            ])

        elif req.mode == "pro_arc":
            persona = await wf.execute_activity(casting, intel, brand)
            script = await wf.execute_activity(screenwriter_3act, persona, intel, brand)
            shots = await wf.execute_activity(director, script)
            bible = await wf.execute_activity(continuity, persona, shots, brand)
            unit = await wf.execute_activity(duration_plan, script, shots, bible)
            result = await wf.execute_child_workflow(RenderUnitWorkflow, unit, bible)
            results = [result]

        elif req.mode == "tv":
            concepts = await wf.execute_activity(concept_gen, intel, brand)
            await wf.wait_condition(lambda: self.concept_choice is not None)
            script = await wf.execute_activity(screenwriter_broadcast, self.concept_choice, intel, brand)
            shots = await wf.execute_activity(director, script)
            await wf.wait_condition(lambda: self.storyboard_approved)
            bible = await wf.execute_activity(continuity, None, shots, brand)
            units = [await wf.execute_activity(duration_plan, shot) for shot in shots]
            shot_results = await asyncio.gather(*[
                wf.execute_child_workflow(RenderUnitWorkflow, u, bible) for u in units
            ])
            vo, music = await asyncio.gather(
                wf.execute_activity(vo_agent, script, persona_card_from(bible)),
                wf.execute_activity(composer, script, brand),
            )
            results = shot_results + [vo, music]

        final = await wf.execute_activity(editor, results, req)
        qa = await wf.execute_activity(qa_agent, final, brand)
        if not qa.passed:
            # trigger targeted regen path
            ...
        return final
```

`RenderUnitWorkflow` is the per-unit child workflow that handles single_gen / extend_chain / cut_chain internally:

```python
@workflow.defn
class RenderUnitWorkflow:
    @workflow.run
    async def run(self, unit: RenderUnit, bible: StyleBible | None = None) -> UnitResult:
        if unit.pattern == "single_gen":
            seg = unit.segments[0]
            plan = route_segment(seg, unit, bible)
            return await wf.execute_activity(video_generate, seg, plan)

        if unit.pattern == "cut_chain":
            results = await asyncio.gather(*[
                wf.execute_activity(video_generate, seg, route_segment(seg, unit, bible))
                for seg in unit.segments
            ])
            return stitch_cut_chain(results)

        if unit.pattern == "extend_chain":
            prev = await wf.execute_activity(video_generate, unit.segments[0], route_segment(unit.segments[0], unit, bible))
            for seg in unit.segments[1:]:
                plan = route_segment(seg, unit, bible)
                plan.source_video_url = prev.video_url
                prev = await wf.execute_activity(video_generate, seg, plan)
            return prev  # final extended video
```

Extend-chains are sequential by necessity. Cut-chains fan out for parallel rendering.

## 10. Per-Mode Pipeline Walkthroughs

### 10.1 Mode A — UGC, ~3–6 min

1. Upload image + name → job created, workflow started.
2. Product Intel (GPT-4o vision) ≈ 5s.
3. Brand Strategist (cheap, skipped if no brand) ≈ 3s.
4. Casting Director ≈ 10s.
5. Screenwriter generates 3 scripts in parallel ≈ 20s.
6. Duration Planner tags each script as single_gen/15s ≈ 2s.
7. For each of 3 scripts, in parallel:
   - Frame Designer (Gemini 2.5 Flash Image) ≈ 15s.
   - Video Router picks Seedance 2.0.
   - Seedance generation via Kie.ai ≈ 90–180s (poll).
8. Editor: caption burn-in (WhisperX transcript → styled subs), 9:16 export, optional 1:1 center crop.
9. QA Agent.
10. Websocket emits `job_completed`. 3 videos in the dashboard.

### 10.2 Mode B — Professional UGC, ~5–12 min

- **B1 (Continuous, Seedance 2.0 extend):** Screenwriter writes one 3-act 30s script → Director produces a single shot with 2 temporal beats → Duration Planner emits one `extend_chain` unit with 2 segments → segment 0 on Seedance base, segment 1 on Seedance extend with `source_video_url` = segment 0's output. One final 30s file, no visible cuts.
- **B2 (Cut-Based, Seedance):** Script → Director produces 2–3 shots with location/time jumps → Continuity Manager emits a Style Bible + character reference card → each shot: Frame Designer conditioned on bible → Seedance generation. Editor stitches with matched color grade pass so cuts feel intentional, not jarring.

The Director decides B1 vs B2 as part of its output. Override in request.

### 10.3 Mode C — TV Commercial, ~15–30 min with two human approvals

1. Brand Audit pulls palette/voice/past-ad embeddings from pgvector.
2. Concept Generator (Opus 4.7) returns 3 one-page treatments.
3. **Workflow paused** on `wait_condition`. User picks one in the UI.
4. Screenwriter (Broadcast overlay) → Director → Storyboard Artist generates storyboard with Frame Designer drafts of each shot.
5. **Workflow paused again** on storyboard approval. User can edit any shot's direction in the scene editor before committing to rendering.
6. On approval: Continuity Manager emits Style Bible. For each shot, Duration Planner decides single_gen vs (rare) short extend_chain.
7. Shots render in parallel (Temporal child workflows). Router picks Seedance as default, Veo 3 for shots marked cinematic in the bible.
8. VO Agent and Composer run in parallel with shots.
9. Editor: assemble master, auto-derive 6s/15s/60s cutdowns by selecting beats the Director tagged as "key," render 16:9 + 9:16 + 1:1.
10. QA Agent.

## 11. Key Engineering Challenges & Solutions

### 11.1 Continuity — now much easier

Extend carries ~80% of the continuity weight for Mode B1 and any long hero shot. For cut-chain continuity (Mode B2 and Mode C), the stack is:

1. **Character reference card.** At casting time, generate a canonical 3/4-view portrait via Gemini 2.5 Flash Image. Becomes a reference in every downstream Frame Designer call.
2. **Style Bible.** Text-level lock on wardrobe, hair, lighting direction, color palette, forbidden elements. Prepended to every per-shot prompt.
3. **First-frame chaining.** Shot N's first frame is Gemini-edited from shot N−1's last frame to inherit visual DNA.
4. **Video model choice.** For cross-cut shots with a recurring character, prefer the model that handles reference images best (currently Veo 3; verify at build time). Pay the cost for character-critical shots; use Seedance for establishing/B-roll shots.
5. **Scene editor fallback.** When auto-continuity fails, the user can lock a shot and regen adjacent shots until it matches.

### 11.2 Cost control

This is where the 15s constraint plus Seedance default pays us back.

- **Seedance default.** Most shots in most spots are Seedance — materially cheaper than Veo 3. Verify current prices before launch, but the structural gap is consistent.
- **Veo 3 earned, not assumed.** Only high-polish shots get Veo 3.
- **Hook Lab.** 3s opener variants on Seedance before committing the full render. Saves 60–80% of A/B cost.
- **Anthropic prompt caching** on Casting + Screenwriter + Director system prompts cuts LLM cost ~70% on repeat-structure runs.
- **Per-segment regen**, not per-job. Editing shot 3 doesn't re-render shots 1–2.
- **Pre-commit cost quote.** UI shows estimated credits before the user hits render. No bill shock.
- **Polish slider → cost slider.** Higher polish = more Veo 3 = more credits. Transparent to the user.

### 11.3 Latency and async UX

- Never block page load on a job. Submit → redirect.
- Websocket granular events: `intel_ready`, `persona_ready`, `script_ready`, `frame_1_ready` (show the still immediately — perceived progress), `segment_1_rendering`, `segment_1_ready`, etc.
- Email on job complete in case tab was closed.
- Per-provider concurrency limits enforced at the Temporal activity level (e.g., 5 concurrent Veo 3 calls, 10 concurrent Seedance). Overflows queue, don't fail.

### 11.4 Multi-provider routing + fallback

```python
async def execute_with_fallback(seg: Segment, plan: ModelPlan) -> VideoResult:
    last_error = None
    for model in [plan.model] + plan.fallback:
        try:
            client = get_client(model)
            job = await client.generate(seg, plan)
            result = await client.poll(job, timeout=300)
            if passes_basic_qa(result):
                return result
        except (RateLimit, ProviderDown, QAFail) as e:
            last_error = e
            continue
    raise PipelineFailure(last_error)
```

Fallback ladder per model family is defined at the router. Seedance→Veo 3 Fast→Kling. Veo 3→Kling→Seedance. Extend cannot fall back to a non-extending model; if Seedance extend is down, we degrade gracefully to cut-chain mode and notify the user.

### 11.5 Prompt evaluation & versioning

- All prompts in `/prompts/<agent>/v<n>.jinja`, versioned in Git.
- Eval suite: 20 diverse products × 3 demographics each. Runs automatically on PR.
- LLM-judge scoring (Opus 4.7 as judge against a rubric: specificity, on-brief adherence, non-goal compliance, format correctness).
- Human review on 10% sampled outputs weekly.
- A prompt version promotes to prod only if it wins ≥70% vs the incumbent and regresses on nothing.
- Jobs-in-flight keep their prompt version (reproducibility).

### 11.6 Brand safety

- Screenwriter receives ONLY structured product intel, persona, brand constraints. Cannot invent prices, claims, ingredients, URLs.
- `brand.forbidden_claims` enforced post-gen by QA Agent (substring + semantic embedding match).
- Regulated categories (supplements, finance, medical) load a category rules pack; QA blocks on violation.
- No real people, no celebrities, no competitor brand mentions — pre-gen classifier on every script.

- **Prompts in-repo, versioned in Git** — zero prompt-management tool cost.
- **Fly.io for v1** — no EKS setup tax until there's traffic to justify it.

## 15. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Kie.ai or fal.ai API instability | Multi-provider router from day one; Kling and Runway as fallback ladder. Graceful degradation (extend → cut-chain). |

| Cost runaway on power users | Hard credit caps at the plan limit; top-ups required beyond that; pre-commit cost quote in UI. |
| Agency data-residency / security review | SOC 2 roadmap in Phase 4; per-workspace isolation from day 1. |

## Appendix A — What we're NOT using from the inspiration workflow

- **n8n.** Temporal fits 20-minute multi-branch jobs with replayability and observability.
- **Form trigger.** Full product UI: brand mgmt, scene editor, job history.
- **Single-mode assumption.** Mode-conditioned agent graphs.
- **Hardcoded Sora 2.** It's deprecated — we're on Seedance 2.0 extend via Kie.ai, with Veo 3 via fal.ai for selective cinematic shots.
- **Black-box Drive delivery.** Platform-aware exports (safe zones, aspects, cutdowns).
- **Single-shot assumption.** Duration Planner + render units + multi-shot with continuity.

## Appendix B — What we ARE adopting (the crown jewels)

- Casting Director → Screenwriter separation of concerns.
- Rigid, numbered output structure in the Casting prompt.
- WANT / AVOID framing blocks.
- Per-second shot breakdowns (extended to per-shot for Modes B/C, per-segment for extend-chains).
- "NO invented details" discipline in every creative agent.
- First-frame-before-video pattern (product-anchored first frame as visual condition for the video model).

## Appendix C — The efficiency principles, one-line each

1. **Cheapest path that meets the quality bar.** Seedance default; Veo 3 earned.
2. **Typed I/O between agents.** Kills whole classes of bugs.
3. **Render units as the duration primitive.** Unifies single-gen, extend-chain, cut-chain.
4. **Regen at the lowest viable granularity.** Segment > unit > job.
5. **Prompts in Git, versioned, evaluated.** Changes are reviewable.
6. **Replayable pipeline via agent_output storage.** Iteration is free.
7. **Cache before compute.** Prompt caching, frame caching, style bible reuse.
8. **Fan out where possible, sequence only where forced.** Cut-chains parallelize; extend-chains don't.
9. **Fail over, don't fail out.** Router ladder + graceful degradation.
10. **Ship Mode A in 5 weeks.** Revenue starts the flywheel.
